1. Proyecto Pygame Shader¶

PygameShader es un envoltorio alrededor de la biblioteca Pygame, escrito en Python, Cython y C, diseñado para agregar efectos visuales avanzados a aplicaciones multimedia, incluyendo juegos de video indie en 2D, juegos de arcade y procesamiento de imágenes en tiempo real para videos y transmisiones de cámaras.

Esta biblioteca extiende Pygame permitiendo a los desarrolladores aplicar shaders o procesadores de texturas a texturas de sprites, superficies SDL, y a la pantalla completa del juego, mejorando significativamente la experiencia visual con una sobrecarga mínima de rendimiento.

Formatos de Superficie/Imagen Soportados

PygameShader soporta múltiples formatos de imagen, incluyendo BMP, GIF (no animado), JPEG y PNG. Sin embargo, Pygame puede no estar siempre compilado con soporte para todos los formatos. Al menos, BMP siempre está disponible.

Para comprobar si se soportan formatos adicionales de imagen, ejecuta:

import pygame
print(pygame.image.get_extended())

Si devuelve True, entonces los formatos PNG, JPG y GIF también están soportados.

Rendimiento y Optimización

PygameShader está optimizado para renderizado en tiempo real a 60 FPS, particularmente para resoluciones medias (1024x768). Sin embargo, el rendimiento depende de la complejidad del shader.

Algunos efectos, como filtros medianos, visión de depredador y efecto de dibujo animado, requieren múltiples shaders para crear efectos compuestos, lo que los hace más exigentes computacionalmente. La mayoría de los shaders mantendrán un rendimiento fluido a resoluciones medias, pero resoluciones más altas pueden afectar las tasas de fotogramas.

Para texturización de sprites y efectos especiales, PygameShader sigue siendo extremadamente rápido, gracias a las optimizaciones basadas en Cython. Para mantener una tasa de fotogramas alta, se recomienda mantener el tamaño de las texturas de los sprites SDL dentro de 200x200 píxeles para evitar sobrecargas de procesamiento innecesarias.

Características

  • Efectos de Shader – Mejora los visuales de tu juego con técnicas de sombreamiento avanzadas.

  • Renderizado en tiempo real – Aplica efectos a la pantalla completa del juego a altas tasas de fotogramas.

  • Optimizado para Velocidad – La implementación eficiente de Cython asegura un rendimiento fluido.

  • Personalización de Sprites y Superficies – Modifica texturas y superficies dúctilmente.

PygameShader proporciona herramientas poderosas para mejorar el atractivo visual de tu juego, ya sea mejorando texturas de sprites o aplicando efectos visuales a pantalla completa.

2. Shaders de GPU con CuPy y CUDA¶

Algunos scripts han sido portados a la GPU usando CuPy y kernels crudos de CUDA, permitiendo que se ejecuten en tarjetas gráficas NVIDIA (capacidad de computación 3.0 o superior).

Advertencia

Estos shaders son compatibles solo con chipsets NVIDIA.

Requisitos

Para usar los shaders de GPU, debes tener:

  • La biblioteca CuPy instalada.

  • El CUDA Toolkit instalado (versiones soportadas: v10.2 - v11.6).

Para instrucciones de instalación, consulta la guía de instalación de CuPy.

Estado Experimental y Consideraciones de Rendimiento

Los shaders de GPU son experimentales y pueden estar limitados por el ancho de banda PCI Express, especialmente para renderizado en tiempo real.

Para un ejemplo de renderizado en tiempo real de GPU de un pygame.Surface, consulta:

Demo/GPU_demo_ripple.py

Comparación: GLSL vs. CUDA

Si bien GLSL supera a CuPy/CUDA en el rendimiento gráfico bruto, CUDA sobresale en acelerar cálculos paralelos de propósito general originalmente pensados para arquitecturas de CPU (Python y Cython).

3. Instalación¶

Instalar vía pip

Verifica la última versión: PygameShader en PyPI

pip install PygameShader

Versiones de Python soportadas: 3.6 - 3.11 Plataformas: Windows & Linux (x86, x86_64)

Verifica la versión instalada:

from PygameShader.shader import __VERSION__
print(__VERSION__)

Instalar desde el código fuente

Linux¶

tar -xvf source-1.0.8.tar.gz
cd PygameShader-1.0.8
python3 setup.py bdist_wheel
cd dist
pip3 install PygameShader-xxxxxx

Windows¶

python setup.py bdist_wheel
cd dist
pip install PygameShader-xxxxxx

Compilando Cython y Código C

¿Cuándo debes compilar el código Cython?

Si modificas algún archivo .pyx o .pxd (por ejemplo, shader.pyx, shader.pxd, __init__.pxd), recompila usando:

python setup_shader.py build_ext --inplace --force

Para una versión específica de Python, usa:

python3.8 setup_shader.py build_ext --inplace

Si la compilación falla, asegúrate de:

  • Cython y un compilador C (MSVC, MinGW, GCC) están instalados.

  • Los encabezados de desarrollo de Python están disponibles.

Para usuarios de Windows, consulta esta guía: Configuración del compilador VC++.

4. OpenMP para procesamiento multihilo¶

Por defecto, PygameShader se compila con soporte para multiprocesamiento (OpenMP).

Para deshabilitar OpenMP, actualiza config.py:

# Enable/disable multi-processing
OPENMP = False
OPENMP_PROC = "-fopenmp"

Luego reconstruye:

python setup.py bdist_wheel
cd dist
pip install PygameShader-xxxxxx.whl
