Misc¶

Misc.pyx


1. Résumé de la bibliothèque Python¶

Cette bibliothèque fait partie d’un module Python pour la manipulation et le traitement d’images, axé sur la génération de gradients, la manipulation des canaux d’images, les calculs de l’espace colorimétrique, et les algorithmes de tri. Voici un aperçu des principales fonctionnalités et objectifs des différentes fonctions et méthodes du module :

Fonctions de génération de gradients

Ces fonctions créent divers types de gradients, souvent utilisés dans les applications graphiques pour créer des transitions douces entre les couleurs.

  • `create_line_gradient_rgb` : Génère un gradient linéaire de valeurs RGB entre deux couleurs spécifiées (start_rgb et end_rgb).

  • `create_line_gradient_rgba` : Similaire à la précédente, mais pour les couleurs RGBA (ajoute un canal alpha pour la transparence).

  • `horizontal_rgb_gradient` : Crée un gradient horizontal dans l’espace colorimétrique RGB.

  • `horizontal_rgba_gradient` : Crée un gradient horizontal dans l’espace colorimétrique RGBA.

  • `create_radial_gradient` : Crée un gradient radial qui rayonne à partir d’un point, avec des couleurs de début et de fin personnalisables et des décalages. Un gradient pré-calculé peut être fourni pour optimiser les performances.

  • `create_radial_gradient_alpha` : Similaire à create_radial_gradient, mais spécifiquement pour gérer les RGBA avec transparence.

  • `create_quarter_radial_gradient` et `create_quarter_radial_gradient_alpha` : Ces fonctions créent des gradients pour un quart de cercle (gradients sectoriels), utiles pour des effets visuels spécialisés.

Fonctions de manipulation d’espace colorimétrique et d’appariement de couleurs

Ces fonctions sont utilisées pour travailler avec différents espaces colorimétriques (HSV, HSL, RGB) et trouver les couleurs les plus proches, ce qui est utile pour le traitement d’images basé sur les couleurs ou la génération de palettes.

  • `color_dist_hsv` : Calcule la distance entre deux couleurs dans l’espace colorimétrique HSV.

  • `color_dist_hsl` : Calcule la distance entre deux couleurs dans l’espace colorimétrique HSL.

  • `closest_hsv_color` et `closest_hsl_color` : Trouve la couleur la plus proche dans une palette donnée par rapport à une couleur cible, en utilisant respectivement les espaces colorimétriques HSV ou HSL.

  • `close_color` : Trouve une couleur dans une palette proche d’une couleur donnée (probablement en utilisant l’espace RGB).

  • `use_palette` : Similaire à close_color, mais sélectionne des couleurs dans une palette basée sur un critère de sélection défini.

Fonctions de manipulation d’image

Ces fonctions se concentrent sur la manipulation d’images ou de tampons, en particulier pour le défilement, le mélange et la combinaison des canaux.

  • `swap_image_channels` : Échange les canaux de couleur (comme RGB) sur une surface d’image, utile lorsqu’on travaille avec différents formats de couleur ou qu’on effectue des transformations.

  • `scroll_surface_24bit`, `scroll24_inplace`, `scroll_rgb_array_inplace`, `scroll_alpha_inplace` : Ces fonctions permettent de faire défiler une image ou des canaux d’image dans les directions x et y, ce qui peut être utilisé pour des animations ou des effets visuels. Les versions c sont probablement des implémentations optimisées écrites en C pour les performances.

  • `combine_rgba_buffers` : Combine un tampon d’image BGR et un canal alpha en un seul tampon d’image RGBA. Cela est utile pour travailler avec des images ayant des canaux de couleur et alpha séparés.

  • `normalize_2d_array` : Normalise un tableau 2D, probablement pour la normalisation des couleurs ou de l’intensité des pixels.

Fonctions utilitaires

Ces fonctions accomplissent diverses tâches utilitaires, y compris le tri et la génération de nombres aléatoires.

  • `rgb_to_int` et `int_to_rgb` : Convertit entre les valeurs RGB et leur représentation entière (souvent utilisée pour l’emballage/déballage des couleurs).

  • `_randf`, `_randi` : Génère des valeurs aléatoires en float et en entier dans une plage spécifiée, probablement pour la génération aléatoire de couleurs ou de pixels.

  • `bubble_sort`, `insertion_sort`, `quick_sort`, `heap_sort` : Ce sont différents algorithmes de tri implémentés pour trier des tableaux de données de pixels ou d’images. heap_sort et quick_sort sont particulièrement utiles pour trier de grands ensembles de données ou des valeurs de pixels de manière efficace.

  • `partition_cython`, `_quick_sort`, `heapify` : Ce sont des fonctions d’aide pour les algorithmes de tri comme QuickSort et HeapSort, optimisées pour les performances en Cython.

Vérification du format et du type d’image

Ces fonctions aident à déterminer le type de données ou de tampons d’image.

  • `get_image_format` : Vérifie probablement le format d’une image (si elle est en RGBA, RGB, etc.).

  • `is_type_memoryview`, `is_uint8`, `is_float64`, `is_int32` : Ces fonctions vérifient le type et le format des données d’un tableau ou tampon d’image donné, utile pour valider l’entrée et garantir le bon format de données.

2. Objectif global de la bibliothèque¶

La bibliothèque est conçue pour le traitement d’images et les opérations graphiques, en mettant l’accent sur la création de gradients, la manipulation des canaux d’image, les transformations de couleurs, les algorithmes de tri, et l’optimisation. Elle fournit des fonctions pour créer des transitions de couleurs douces (gradients), manipuler les données de pixels, vérifier les formats d’images, et optimiser les opérations coûteuses en calcul en utilisant des algorithmes comme QuickSort et HeapSort. Les fonctions sont fortement optimisées avec cpdef et cdef pour permettre une intégration fluide entre Python et C pour les tâches critiques en termes de performance.

3. Cas d’utilisation¶

  • Traitement d’image : Manipulation et transformation des images, gestion des canaux RGBA/BGR, défilement des pixels d’image et application de transformations de couleur.

  • Rendu graphique : Génération de gradients (linéaires, radiaux, quart de cercle) pour des effets de fond ou des transitions visuelles.

  • Correspondance des couleurs : Trouver les couleurs les plus proches dans une palette donnée en fonction de différents espaces colorimétriques (HSV, HSL).

  • Optimisation des performances : Utilisation d’algorithmes de tri et de manipulation des couleurs pour traiter efficacement de grandes images ou ensembles de données, avec des améliorations de performance grâce à Cython (cpdef/cdef).

4. Conclusion¶

Cette bibliothèque serait utile dans des applications telles que les éditeurs d’images, les outils de conception graphique, la visualisation de données ou les applications graphiques en temps réel.

5. Méthodes de liste Cython¶


Misc.swap_image_channels(image_surface, channel_order)¶

Résumé : Échange ou annule les canaux d’une image en fonction de la chaîne d’ordre des canaux spécifiée. La fonction permet d’échanger les canaux de couleur (RGB) d’une image représentée comme un pygame.Surface. Vous pouvez spécifier le nouvel ordre des canaux à l’aide d’une chaîne où chaque lettre représente un canal de couleur (R, G, B) ou “0” pour annuler un canal (le supprimer).

Exemple d’utilisation :

new_surface = swap_image_channels(image_surface, 'R0B')
# Swaps red and blue channels and removes green

Paramètres :

  • image_surface (pygame.Surface) :

    L’image (pygame Surface) dont les canaux doivent être échangés.

  • channel_order (str) :

    Une chaîne représentant l’ordre souhaité des canaux. Elle doit contenir exactement 3 caractères, où “R”, “G”, et “B” représentent les canaux de couleur respectifs, et “0” annule un canal. Exemples de valeurs : “RGB”, “RBG”, “GRB”, “BGR”, “BRG”, “R0B”, etc.

Retourne : Une nouvelle surface pygame 24 bits avec les canaux échangés.

Lève : - ValueError :

Si la chaîne d’ordre des canaux n’est pas valide ou si le format de la surface est incompatible.


Misc.create_line_gradient_rgb(num_pixels, start_rgb=(255, 0, 0), end_rgb=(0, 255, 0))¶

Résumé : Génère un gradient horizontal 2D de couleurs RGB allant d’une couleur de départ à une couleur de fin. La fonction crée un tableau de valeurs RGB représentant un gradient horizontal lisse transitionnant de start_rgb à end_rgb. Le gradient consiste en num_pixels pixels, avec la couleur changeant progressivement de la couleur de départ à la couleur de fin à travers le tableau 1D.

Exemple d’utilisation :

gradient = create_line_gradient_rgb(256, start_rgb=(255, 0, 0), end_rgb=(0, 0, 255))
# Generates a horizontal gradient from red to blue with 256 pixels.

Paramètres :

  • num_pixels (int) :

    Le nombre de pixels dans le gradient (doit être > 0). Détermine la longueur du gradient dans la direction horizontale.

  • start_rgb (tuple, optionnel) :

    Un tuple représentant les valeurs RGB de la couleur de départ (par défaut, rouge : (255, 0, 0)).

  • end_rgb (tuple, optionnel) :

    Un tuple représentant les valeurs RGB de la couleur de fin (par défaut, vert : (0, 255, 0)).

Retourne : - numpy.ndarray :

Un tableau numpy 2D de forme (num_pixels, 3), où chaque ligne représente une couleur RGB. Le tableau contient les valeurs des pixels du gradient, avec la couleur transitionnant de start_rgb à end_rgb.

Lève : - ValueError :

Si num_pixels est inférieur ou égal à 0. Si start_rgb ou end_rgb ne sont pas des tuples RGB valides de longueur 3, ou si l’une des valeurs de couleur est en dehors de la plage RGB valide [0, 255]. Si num_pixels n’est pas un entier positif.

  • TypeError :

    Si start_rgb ou end_rgb ne sont pas des tuples, ou s’ils contiennent des valeurs non entières.

Misc.horizontal_rgb_gradient(w, h, color_start=(255, 0, 0), color_end=(0, 255, 0))¶

Résumé : Génère un gradient horizontal 24 bits entre deux couleurs RGB. La fonction crée une transition douce de color_start à color_end à travers une surface de dimensions spécifiées.

Exemple d’utilisation :

# Create a gradient from red to green
gradient = horizontal_rgb_gradient(500, 300, (255, 0, 0), (0, 255, 0))

Paramètres :

  • w (int) :

    La largeur de la surface du gradient en pixels (doit être > 1).

  • h (int) :

    La hauteur de la surface du gradient en pixels (doit être > 0).

  • color_start (tuple, optionnel) :

    La couleur de départ sous forme de tuple (R, G, B), où chaque valeur est dans la plage [0, 255]. La valeur par défaut est (255, 0, 0), représentant le rouge.

  • color_end (tuple, optional):

    La couleur finale sous forme de tuple (R, G, B), où chaque valeur est dans l’intervalle [0, 255]. La valeur par défaut est (0, 255, 0), ce qui représente le vert.

Retour : - pygame.Surface :

Un objet pygame.Surface RGB 24 bits de taille (w, h) avec le dégradé appliqué.

Lève : - ValueError :

Si w ou h n’est pas valide. Si color_start ou color_end n’est pas un tuple RGB valide.


Misc.horizontal_rgba_gradient(w, h, color_start=(255, 0, 0, 255), color_end=(0, 255, 0, 0))¶

Résumé : Génère un dégradé horizontal 32 bits avec transparence entre deux couleurs RGBA. La fonction crée une transition horizontale fluide de color_start à color_end à travers la largeur donnée tout en maintenant la hauteur spécifiée.

Exemple d’utilisation :

# Create a gradient from red to green
gradient = horizontal_rgba_gradient(500, 300, (255, 0, 0, 0), (0, 255, 0, 255))

Paramètres :

  • w (int) :

    La largeur de la surface du gradient en pixels (doit être > 1).

  • h (int) :

    La hauteur de la surface du gradient en pixels (doit être > 0).

  • color_start (tuple, optionnel) :

    La couleur de départ au format RGBA, où chaque composant (R, G, B, A) est dans l’intervalle 0-255. Par défaut : (255, 0, 0, 255) (rouge opaque).

  • color_end (tuple, optional):

    La couleur de fin au format RGBA, où chaque composant (R, G, B, A) est dans l’intervalle 0-255. Par défaut : (0, 255, 0, 0) (vert transparent).

Retour : - pygame.Surface :

Un pygame.Surface 32 bits par pixel avec les dimensions (w, h), contenant le dégradé généré avec un canal alpha.

Lève : - ValueError :

  • Si w ou h n’est pas un entier positif valide.

  • Si color_start ou color_end n’est pas un tuple de quatre entiers (0-255).


Misc.create_radial_gradient(w, h, offset_x=0.5, offset_y=0.5, color_start=(255, 0, 0), color_end=(0, 0, 0), precomputed_gradient=None, scale_factor=r_max, threads=8)¶

Résumé : Génère un dégradé radial (RGB 24 bits) avec une transition fluide entre deux couleurs, centré dans une surface rectangulaire de largeur et hauteur données. Le dégradé rayonne depuis le centre de la surface, et la transition entre les couleurs est basée sur la distance au centre, permettant divers effets de dégradé (tel que circulaire ou radial). Optionnellement, un dégradé pré-calculé peut être utilisé, ou un dégradé peut être généré dynamiquement.

Exemple d’utilisation :

precomputed_gradient = create_line_gradient_rgb(
    math.sqrt(800 ** 2 + 800 ** 2),
    start_rgb = (255, 0, 0),
    end_rgb = (0, 0, 0))

surface = create_radial_gradient(800, 800,
    offset_x = 0.5,
    offset_y = 0.5,
    color_start = (255, 0, 0),
    color_end = (0, 0, 0),
    precomputed_gradient = precomputed_gradient
)

Paramètres :

  • w (int) :

    La largeur de la surface du dégradé de sortie en pixels (doit être > 0).

  • h (int) :

    La hauteur de la surface du dégradé de sortie en pixels (doit être > 0).

  • offset_x (float, optionnel) :

    Le décalage horizontal du centre du dégradé, où 0,5 représente le centre de la surface (par défaut 0,5).

  • offset_y (float, optionnel) :

    Le décalage vertical du centre du dégradé, où 0,5 représente le centre de la surface (par défaut 0,5).

  • color_start (tuple, optionnel) :

    La couleur de départ du dégradé au format RGB (0-255). Par défaut, (255, 0, 0), ce qui correspond au rouge opaque.

  • color_end (tuple, optional):

    La couleur de fin du dégradé au format RGB (0-255). Par défaut : (0, 0, 0), ce qui correspond au noir.

  • precomputed_gradient (numpy.array, optionnel) :

    Un tableau 2D de dégradé pré-calculé (forme : (n, 3), contenant les valeurs RGB). Si non fourni, un nouveau dégradé sera calculé.

  • scale_factor (float, optionnel) :

    Un facteur de mise à l’échelle qui ajuste le rayon du dégradé. Une valeur supérieure à 1 augmentera la taille du dégradé, et des valeurs inférieures à 1 réduiront son rayon. Par défaut : 1,4. Doit être > 0.

  • threads (int, optionnel) :

    Le nombre de threads simultanés à utiliser pour le calcul du dégradé. Par défaut 8.

Retour : - pygame.Surface :

Un objet pygame.Surface avec le dégradé radial généré, centré à (w/2, h/2), au format RGB 24 bits.

Lève : - ValueError :

  • Si color_start ou color_end n’est pas un tuple de 3 entiers.

  • Si une valeur RGB dans color_start ou color_end est hors de portée (0-255).

  • Si scale_factor est None ou <= 0.

  • Si w ou h est <= 1.

  • TypeError :

    • Si precomputed_gradient n’est pas un tableau contigu.

  • ValueError :

    • Si precomputed_gradient n’est pas un tableau 2D, n’est pas de type uint8, a une longueur <= 1, ou n’est pas au format RGB.

Crédits : Fonction améliorée avec la collaboration de ChatGPT.


Misc.create_radial_gradient_alpha(w, h, offset_x=0.5, offset_y=0.5, color_start=(255, 0, 0, 255), color_end=(0, 0, 0, 0), precomputed_gradient=None, scale_factor=r_max, threads=8)¶

Résumé : Crée un dégradé radial 32 bits avec transparence, fusionnant en douceur entre deux couleurs RGBA (color_start et color_end), et prend en charge les tableaux de dégradé pré-calculés pour l’optimisation des performances.

Paramètres :

  • w (int) : La largeur de la surface en pixels (doit être > 1).

  • h (int) : La hauteur de la surface en pixels (doit être > 1).

  • offset_x (float, optionnel) : La coordonnée X du centre du dégradé en fraction de la largeur (par défaut 0.5).

  • offset_y (float, optionnel) : La coordonnée Y du centre du dégradé en fraction de la hauteur (par défaut 0.5).

  • color_start (tuple, optionnel) : La couleur RGBA de départ du dégradé (par défaut (255, 0, 0, 255)).

  • color_end (tuple, optionnel) : La couleur RGBA de fin du dégradé (par défaut (0, 0, 0, 0)).

  • precomputed_gradient (numpy.ndarray ou None, optionnel) : Un tableau de dégradé pré-calculé de forme (w, 4) au format uint8 pour l’optimisation (par défaut None).

  • scale_factor (float, optionnel) : Le facteur de mise à l’échelle pour le dégradé (doit être > 0, la valeur par défaut est r_max).

  • threads (entier court non signé, optionnel) : Le nombre de threads à utiliser pour le calcul parallèle (la valeur par défaut est 8).

Retourne : - pygame.Surface : Une surface contenant le dégradé radial généré avec une transparence par pixel.

Lève : - ValueError : Si color_start ou color_end n’est pas un tuple de quatre entiers, si une valeur RGBA est hors de la plage (0-255), ou si scale_factor est None ou <= 0. - ValueError : Si w ou h est <= 1. - TypeError : Si precomputed_gradient n’est pas un tableau contigu. - ValueError : Si precomputed_gradient n’est pas un tableau 2D, n’est pas de type uint8, a une longueur <= 1, ou n’est pas en format RGBA.

Exemple d’utilisation :

precomputed_gradient = create_line_gradient_rgba(
    int(math.sqrt(800 ** 2 + 800 ** 2)),
    start_rgba=(255, 0, 0, 255),
    end_rgba=(0, 0, 0, 0)
)

surface = create_radial_gradient_alpha(
    800, 800,
    precomputed_gradient=precomputed_gradient
).convert_alpha()

Misc.create_quarter_radial_gradient(width_, height_, start_color_=(255, 0, 0), end_color_=(0, 0, 0), gradient_array_=None, factor_=1.4, threads_=8)¶

Résumé : Génère un dégradé radial d’un quart (24 bits opaque). Cette fonction crée un dégradé radial en itérant sur le quart nord-ouest (NW) de la surface (width_/2, height_/2) et en miroir les pixels calculés dans les trois autres quadrants (nord-est (NE), sud-est (SE) et sud-ouest (SW)).

Le dégradé passe de start_color_ au centre à end_color_ sur les bords.

Paramètres :

  • width_ (int) : La largeur de la surface en pixels (doit être > 1).

  • height_ (int) : La hauteur de la surface en pixels (doit être > 1).

  • start_color_ (tuple, optionnel) : La couleur RGB de départ au centre du dégradé. Par défaut, c’est (255, 0, 0) [Rouge].

  • end_color_ (tuple, optionnel) : La couleur RGB de fin sur le bord extérieur du dégradé. Par défaut, c’est (0, 0, 0) [Noir].

  • gradient_array_ (numpy.ndarray ou None, optionnel) : Un tableau NumPy de forme (width_, 3), contenant des valeurs de couleur RGB précalculées (uint8). Si None, le dégradé est calculé dynamiquement.

  • factor_ (float, optionnel) : Contrôle l’intensité et l’étalement du dégradé. Doit être > 0. La valeur par défaut est 1.4.

  • threads_ (int, optionnel) : Le nombre de threads concurrents à utiliser pour le calcul. La valeur par défaut est 8.

Retour : - pygame.Surface :

Un objet pygame.Surface contenant le dégradé radial, centré sur (width_/2, height_/2).

Exemple d’utilisation :

surface = create_quarter_radial_gradient(
    width_=800,
    height_=800,
    start_color_=(255, 0, 0),
    end_color_=(0, 0, 0),
    factor_=1.4
)

Misc.create_quarter_radial_gradient_alpha(width_, height_, start_color_=(255, 0, 0, 255), end_color_=(0, 0, 0, 0), gradient_array_=None, factor_=1.4, threads_=8)¶

Résumé : Génère un dégradé radial d’un quart avec transparence alpha (32 bits). Cette fonction crée un dégradé radial avec transparence en itérant sur le quart nord-ouest (NW) de la surface (width_/2, height_/2) et en miroir les pixels calculés dans les trois autres quadrants (nord-est (NE), sud-est (SE) et sud-ouest (SW)).

Le dégradé passe en douceur de start_color_ au centre à end_color_ sur les bords, avec un mélange alpha pour la transparence.

Paramètres :

  • width_ (int) : La largeur de la surface en pixels (doit être > 1).

  • height_ (int) : La hauteur de la surface en pixels (doit être > 1).

  • start_color_ (tuple, optionnel) : La couleur au centre du dégradé en format RGBA. La valeur par défaut est (255, 0, 0, 255) [Rouge opaque].

  • end_color_ (tuple, optionnel) : La couleur au bord extérieur du dégradé en format RGBA. La valeur par défaut est (0, 0, 0, 0) [Noir entièrement transparent].

  • gradient_array_ (numpy.ndarray ou None, optionnel) : Un tableau NumPy de forme (width_, 4), contenant des valeurs de couleur RGBA précalculées (uint8). Si None, le dégradé est calculé dynamiquement.

  • factor_ (float, optionnel) : Contrôle l’intensité et l’étalement du dégradé. Doit être > 0. La valeur par défaut est 1.4.

  • threads_ (int, optionnel) : Le nombre de threads concurrents à utiliser pour le calcul. La valeur par défaut est 8.

Retour : - pygame.Surface :

Un objet pygame.Surface contenant le dégradé radial, centré sur (width_/2, height_/2), avec transparence alpha pour un fondu en douceur.

Exemple d’utilisation :

precomputed_gradient = create_line_gradient_rgba(
    int(math.sqrt(800 ** 2 + 800 ** 2)),
    start_rgba=(255, 0, 0, 255),
    end_rgba=(0, 0, 0, 0)
)

surface = create_quarter_radial_gradient_alpha(
    width_=800,
    height_=800,
    start_color_=(255, 0, 0, 255),
    end_color_=(0, 0, 0, 0),
    precomputed_gradient=precomputed_gradient
).convert_alpha()

Misc.scroll_surface_24bit(surface, dx, dy)¶

Résumé : Fait défiler une surface Pygame 24 bits ou 32 bits horizontalement et/ou verticalement.

Cette fonction crée et retourne une nouvelle surface 24 bits avec les données de pixels décalées selon les valeurs spécifiées de dx et dy. Les pixels qui dépassent seront déplacés de l’autre côté.

Paramètres :

  • surface (pygame.Surface) : Un objet Surface Pygame en format 24 bits ou 32 bits.

  • dx (int) : Le nombre de pixels à faire défiler horizontalement. - Un `dx` positif déplace la surface vers la droite. - Un `dx` négatif déplace la surface vers la gauche.

  • dy (int) : Le nombre de pixels à faire défiler verticalement. - Un `dy` positif déplace la surface vers le bas. - Un `dy` négatif déplace la surface vers le haut.

Retour : - pygame.Surface :

Un objet pygame.Surface 24 bits nouveau avec l’effet de défilement appliqué.

Lève : - TypeError :

Si surface n’est pas un objet pygame.Surface.

Exemple d’utilisation :

new_surface = scroll_surface_24bit(original_surface, dx=10, dy=-5)

Cela déplace la surface de 10 pixels vers la droite et de 5 pixels vers le haut.


Misc.scroll24_inplace(surface, dx, dy)¶

Summary: Scrolls a 24-bit Pygame surface horizontally and/or vertically in place.

This function modifies the given surface directly by shifting its pixel data based on the specified horizontal (dx) and vertical (dy) displacement. Scrolling wraps around, meaning pixels that move off one edge reappear on the opposite edge.

Paramètres :

  • surface (pygame.Surface) :

    Un objet pygame.Surface en format 24 bits ou 32 bits (compatible avec l’accès aux pixels).

  • dx (int) : Quantité de défilement horizontal. - Les valeurs positives déplacent les pixels vers la droite. - Les valeurs négatives déplacent les pixels vers la gauche.

  • dy (int) : Quantité de défilement vertical. - Les valeurs positives déplacent les pixels vers le bas. - Les valeurs négatives déplacent les pixels vers le haut.

Retourne :

  • None (modifie la surface sur place).

Lève : - TypeError :

Si surface n’est pas un objet pygame.Surface.

Exemple d’utilisation :

scroll24_inplace(image_surface, 1, 0)

Cela déplace la surface de 1 pixel vers la droite sans créer une nouvelle surface.


Misc.scroll_rgb_array_inplace(rgb_array, dx, dy)¶

Résumé : Fait défiler un tableau de pixels RGB 3D horizontalement et/ou verticalement sur place.

Cette fonction modifie directement le rgb_array donné en déplaçant les valeurs de pixels en fonction des décalages horizontaux (dx) et verticaux (dy). Le défilement fait une boucle, ce qui signifie que les pixels déplacés au-delà des limites du tableau réapparaissent de l’autre côté.

Paramètres :

  • rgb_array (numpy.ndarray) : - Un tableau NumPy 3D contenant des valeurs de pixels RGB. - Le tableau doit référencer tous les pixels pour garantir la modification sur place. - Forme attendue : (hauteur, largeur, 3) (format RGB).

  • dx (int, optionnel, défaut=0) : - Quantité de décalage horizontal. - Les valeurs négatives déplacent les pixels à gauche. - Les valeurs positives déplacent les pixels à droite.

  • dy (int, optionnel, défaut=0) : - Quantité de décalage vertical. - Les valeurs négatives déplacent les pixels vers le haut. - Les valeurs positives déplacent les pixels vers le bas.

Retourne : - None (modifie rgb_array sur place).

Lève :

  • TypeError : Si rgb_array n’est pas un numpy.ndarray ou memoryviewslice.

  • ValueError : - Si rgb_array n’est pas un tableau 3D avec la forme (hauteur, largeur, 3). - Si rgb_array.dtype n’est pas numpy.uint8 (format RGB 8 bits attendu). - Si rgb_array est vide (shape[0] == 0).

Notes :

  • Si dx et dy sont tous deux à zéro, la fonction n’effectue aucune opération.

  • Cette fonction est optimisée pour la performance et fonctionne sans copier de données.

  • Le tableau doit être contigu et correctement référencé pour éviter des comportements inattendus.

Exemple d’utilisation :

scroll_rgb_array_inplace(rgb_array, dx=5, dy=-2)

Cela déplace le tableau de 5 pixels vers la droite et de 2 pixels vers le haut.

Misc.scroll32_inplace(surface, dx, dy)¶

Résumé : Fait défiler une surface Pygame 32 bits horizontalement et/ou verticalement sur place.

Cette fonction modifie directement la surface donnée en déplaçant les données des pixels en fonction des décalages horizontaux (dx) et verticaux (dy). Le défilement fait une boucle, ce qui signifie que les pixels déplacés au-delà des limites de la surface réapparaissent de l’autre côté.

Paramètres :

  • surface (pygame.Surface) : - Un objet pygame.Surface avec un format de pixel 32 bits. - Les données de pixels de la surface seront modifiées directement.

  • dx (int, optionnel, défaut=0) : - Quantité de décalage horizontal. - Les valeurs négatives déplacent les pixels à gauche. - Les valeurs positives déplacent les pixels à droite.

  • dy (int, optionnel, défaut=0) : - Quantité de décalage vertical. - Les valeurs négatives déplacent les pixels vers le haut. - Les valeurs positives déplacent les pixels vers le bas.

Retourne : - None (modifie surface sur place).

Exceptions : - TypeError : Si surface n’est pas une pygame.Surface valide.

Notes :

  • Si dx et dy sont tous deux à zéro, la fonction n’effectue aucune opération.

  • Cette fonction garantit une modification efficace sur place sans copier de données.

  • Le défilement est cyclique (bouclage toroïdal), ce qui signifie que les pixels qui sortent par un bord réapparaissent sur le bord opposé.

Exemple d’utilisation :

surface = pygame.Surface((200, 200))
scroll32_inplace(surface, dx=15, dy=-10)  # Moves 15 pixels right and 10 pixels up.

  • Déclarée comme `inline` pour une exécution plus rapide.

  • Fonctionne sans le verrou global de l’interpréteur Python (`nogil`), ce qui la rend sûre pour une exécution parallèle en Cython.

Misc.combine_rgba_buffers(w, h, bgr_buffer, alpha_channel, output_rgba_buffer, transpose_output=False)¶

Résumé : Combine des tampons mémoire séparés BGR et Alpha en un seul tampon RGBA.

Cette fonction prend des tampons mémoire séparés BGR (Bleu-Vert-Rouge) et Alpha, les empile dans un seul tampon RGBA, et transpose éventuellement le résultat. Elle est optimisée à l’aide de Cython et parallélisée avec OpenMP pour améliorer les performances.

Si transpose_output est True, la vue mémoire de sortie est inversée en échangeant les lignes et les colonnes.

Paramètres :

  • w (int) : - La largeur de la texture.

  • h (int) : - La hauteur de la texture.

  • bgr_buffer (memoryview [char non signé [::1]]): - Une vue mémoire 1D continue contenant les valeurs BGR packagées (uint8). - Taille attendue : largeur * hauteur * 3.

  • alpha_channel (memoryview [char non signé [::1]]): - Une vue mémoire 1D continue contenant les valeurs alpha (uint8). - Taille attendue : largeur * hauteur.

  • output_rgba_buffer (memoryview [char non signé [::1]]): - Une vue mémoire 1D préallouée pour stocker les valeurs RGBA résultantes (uint8). - Taille attendue : largeur * hauteur * 4.

  • transpose_output (booléen, optionnel, par défaut=False) : - Si True, le tampon RGBA résultant est transposé (retourné).

Retourne : - memoryview [char non signé [::1]] :

  • Une vue mémoire continue contenant les valeurs de pixels RGBA combinées (uint8).

Lève :

  • ValueError :

    • Si w ou h est non positif.

    • Si la taille de bgr_buffer ne correspond pas à largeur * hauteur * 3.

    • Si la taille de alpha_channel ne correspond pas à largeur * hauteur.

    • Si output_rgba_buffer n’est pas suffisamment grand pour stocker largeur * hauteur * 4.

Notes :

  • Utilise `memcpy` pour un copier de mémoire efficace.

  • Prend en charge la transposition optionnelle du tampon.

  • Conçu pour des applications nécessitant des performances critiques.

Exemple d’utilisation :

rgba_buffer = combine_rgba_buffers(
    w=800, h=600,
    bgr_buffer=bgr_data,
    alpha_channel=alpha_data,
    output_rgba_buffer=output_data
)

Misc.bubble_sort(nums, size)¶

Résumé :

Trie un tableau de caractères non signés (par exemple, octets ou valeurs de pixels) en utilisant l”algorithme de tri à bulles.

Le tri à bulles compare itérativement les éléments adjacents dans le tableau et les échange si nécessaire. Le processus continue jusqu’à ce que le tableau soit entièrement trié. L’algorithme s’arrête lorsqu’aucun échange n’est effectué pendant un passage complet.

Paramètres :

  • nums (unsigned char [::1]) : - Une vue mémoire ou un tableau unidimensionnel de valeurs de caractères non signés à trier. Typiquement, ce tableau représente des valeurs d’octets ou des données de pixels (par exemple, valeurs RVB).

  • taille (int) : - Le nombre d’éléments dans le tableau nums à trier.

Retourne :

  • None : - Cette fonction trie le tableau sur place. Aucune valeur n’est retournée ; le tableau d’entrée est modifié directement.

Exemple d’utilisation :

nums = bytearray([4, 2, 3, 1, 5])
bubble_sort(nums, len(nums))
print(nums)
# Output: bytearray(b'\x01\x02\x03\x04\x05')

Notes : - Il s’agit d’un algorithme de tri sur place, ce qui signifie qu’il modifie le tableau d’origine. - Le tri à bulles n’est pas l’algorithme de tri le plus efficace, mais il est simple à implémenter. - L’algorithme fonctionne en O(n^2) dans les pires et moyennes situations, ce qui le rend moins efficace pour de grands ensembles de données.


Misc.insertion_sort(nums, size)¶

Résumé : Trie un tableau d’éléments de caractères non signés en utilisant l”algorithme de tri par insertion.

Le tri par insertion parcourt le tableau, prend chaque élément un par un, et l’insère dans la portion triée du tableau en déplaçant les autres éléments en conséquence. Cette implémentation utilise le traitement parallèle pour améliorer les performances.

Exemple d’utilisation :

nums = bytearray([5, 2, 9, 1, 5, 6])
insertion_sort(nums, len(nums))

Paramètres :

  • nums (unsigned char [::1]) : - Un tableau numpy 1D ou une vue mémoire de caractères non signés (uint8), représentant le tableau à trier.

  • taille (int) : - La taille du tableau nums. Elle spécifie le nombre d’éléments dans le tableau.

Retourne :

  • None : - Cette fonction trie le tableau sur place. Aucune valeur n’est retournée ; le tableau d’entrée est modifié directement.

Notes : - L’algorithme fonctionne avec une complexité temporelle de O(n^2) dans les pires et moyennes situations, ce qui le rend moins efficace pour de grands ensembles de données. - Le tri par insertion est particulièrement efficace pour les petits ensembles de données ou les tableaux déjà presque triés. - Le traitement parallèle est utilisé dans cette implémentation pour améliorer les performances.

Cette fonction sélectionne un élément pivot et partitionne le tableau d’entrée nums de sorte que tous les éléments inférieurs au pivot sont déplacés vers la gauche, et tous les éléments supérieurs au pivot sont déplacés vers la droite. Elle retourne l’indice du point de partition.

Misc._quick_sort(items, low, high)¶

Résumé : Implémentation du tri rapide sur place pour un tableau de valeurs de caractères non signés.

Cette fonction trie récursivement le tableau items en sélectionnant un pivot et partitionnant le tableau de sorte que les éléments plus petits que le pivot se déplacent vers la gauche et les éléments plus grands se déplacent vers la droite.   • low (unsigned int) : - L’indice de début du sous-tableau à trier.

  • high (unsigned int) : - L’indice de fin du sous-tableau à trier.


Misc.heapify(nums, heap_size, root_index)¶

Résumé : Fonction heapify itérative ultra-optimisée pour maintenir la propriété de tas maximal.

Cette fonction « filtre vers le bas » la valeur à l’indice de racine donné afin que le sous-arbre ancré à cet indice devienne un tas maximal valide. On suppose que les sous-arbres en dessous sont déjà des tas valides. Pour un tri par tas complet, assurez-vous de construire le tas en appelant cette fonction sur tous les nœuds non-feuilles dans l’ordre inverse.

Paramètres :

  • nums (unsigned char [::1]) : - Une vue mémoire 1D de valeurs de caractères non signés, représentant le tas.

  • heap_size (unsigned int) : - Le nombre d’éléments dans le tas.

  • root_index (unsigned int) : - L’indice du nœud racine du sous-arbre à heapifier.


Misc.heap_sort(nums, n)¶

Résumé : Effectue un tri par tas sur place sur un tableau de valeurs de caractères non signés.

Cette fonction construit d’abord un tas maximal à partir du tableau d’entrée, garantissant que l’élément le plus grand se trouve à la racine. Ensuite, elle extrait de manière répétée l’élément maximum (en l’échangeant avec le dernier élément) et re-heapifie le tas réduit pour maintenir les propriétés du tas jusqu’à ce que le tableau entier soit trié.

Complexité temporelle : - Construction du tas : O(n) - Extraction et re-heapification : O(n log n) - Globalement : O(n log n)

Complexité spatiale : O(1) (tri sur place)

Paramètres :

  • nums (unsigned char [::1]) : - Une vue mémoire 1D de valeurs de caractères non signés à trier.

  • n (unsigned int) : - Le nombre d’éléments dans le tableau.
