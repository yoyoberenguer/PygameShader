1. Projet Pygame Shader¶

PygameShader est un wrapper autour de la bibliothèque Pygame, écrit en Python, Cython et C, conçu pour ajouter des effets visuels avancés aux applications multimédias, y compris les jeux vidéo 2D indépendants, les jeux d’arcade et le traitement d’image en temps réel pour les flux vidéo et caméra.

Cette bibliothèque étend Pygame en permettant aux développeurs d’appliquer des shaders ou des processeurs de texture aux textures des sprites, aux surfaces SDL et à l’ensemble de l’affichage du jeu, améliorant considérablement l’expérience visuelle avec une surcharge minimale en performances.

Formats d’image/surface pris en charge

PygameShader prend en charge plusieurs formats d’image, notamment BMP, GIF (non animé), JPEG et PNG. Cependant, Pygame peut ne pas toujours être compilé avec le support de tous ces formats. Au minimum, BMP est toujours disponible.

Pour vérifier si des formats d’image supplémentaires sont pris en charge, exécutez :

import pygame
print(pygame.image.get_extended())

Si la commande retourne True, alors les formats PNG, JPG et GIF sont également pris en charge.

Performances et optimisation

PygameShader est optimisé pour le rendu en temps réel à 60 FPS, en particulier pour les résolutions moyennes (1024x768). Cependant, les performances dépendent de la complexité du shader.

Certains effets, comme le filtrage médian, la vision de prédateur et l’effet cartoon, nécessitent plusieurs shaders pour créer des effets composites, ce qui les rend plus exigeants en calcul. La plupart des shaders maintiendront des performances fluides à des résolutions moyennes, mais les résolutions plus élevées peuvent affecter les taux d’images.

Pour le texturage des sprites et les effets spéciaux, PygameShader reste extrêmement rapide, grâce aux optimisations basées sur Cython. Pour maintenir un taux d’images élevé, il est recommandé de conserver les tailles de texture des sprites SDL dans une limite de 200x200 pixels afin d’éviter une surcharge inutile du traitement.

Caractéristiques

  • Effets de shaders – Améliorez les visuels de votre jeu avec des techniques de shading avancées.

  • Rendu en temps réel – Appliquez des effets à l’ensemble de l’affichage du jeu à des fréquences d’images élevées.

  • Optimisé pour la vitesse – Une implémentation efficace en Cython garantit des performances fluides.

  • Personnalisation des sprites et des surfaces – Modifiez dynamiquement les textures et les surfaces.

PygameShader propose des outils puissants pour améliorer l’esthétique visuelle de votre jeu, que ce soit en améliorant les textures des sprites ou en appliquant des effets visuels plein écran.

2. Shaders GPU avec CuPy et CUDA¶

Certains scripts ont été portés sur le GPU en utilisant CuPy et les noyaux bruts CUDA, leur permettant de fonctionner sur des cartes graphiques NVIDIA (Compute Capability 3.0 ou supérieur).

Avertissement

Ces shaders sont compatibles uniquement avec les chipsets NVIDIA.

Pré-requis

Pour utiliser les shaders GPU, vous devez avoir :

  • La bibliothèque CuPy installée.

  • Le CUDA Toolkit installé (versions supportées : v10.2 - v11.6).

Pour les instructions d’installation, consultez le guide d’installation de CuPy.

Statut expérimental et considérations de performance

Les shaders GPU sont expérimentaux et peuvent être limités par la bande passante PCI Express, en particulier pour le rendu en temps réel.

Pour un exemple de rendu GPU en temps réel d’un pygame.Surface, consultez :

Demo/GPU_demo_ripple.py

Comparaison : GLSL vs. CUDA

Bien que GLSL surpasse CuPy/CUDA en termes de performances graphiques brutes, CUDA excelle dans l’accélération des calculs parallèles généraux, initialement conçus pour les architectures CPU (Python & Cython).

3. Installation¶

Installation via pip

Vérifiez la dernière version : PygameShader sur PyPI

pip install PygameShader

Versions Python prises en charge : 3.6 - 3.11 Plateformes : Windows & Linux (x86, x86_64)

Vérifiez la version installée :

from PygameShader.shader import __VERSION__
print(__VERSION__)

Installer à partir du code source

Linux¶

tar -xvf source-1.0.8.tar.gz
cd PygameShader-1.0.8
python3 setup.py bdist_wheel
cd dist
pip3 install PygameShader-xxxxxx

Windows¶

python setup.py bdist_wheel
cd dist
pip install PygameShader-xxxxxx

Compilation du code Cython & C

Quand devez-vous compiler le code Cython ?

Si vous modifiez des fichiers .pyx ou .pxd (par exemple, shader.pyx, shader.pxd, __init__.pxd), recompilez en utilisant :

python setup_shader.py build_ext --inplace --force

Pour une version spécifique de Python, utilisez :

python3.8 setup_shader.py build_ext --inplace

Si la compilation échoue, assurez-vous que :

  • Cython & un compilateur C (MSVC, MinGW, GCC) sont installés.

  • Les en-têtes de développement Python sont disponibles.

Pour les utilisateurs de Windows, consultez ce guide : Configuration du compilateur VC++.

4. OpenMP pour le traitement multi-thread¶

Par défaut, PygameShader est compilé avec le support du traitement parallèle (OpenMP).

Pour désactiver OpenMP, mettez à jour config.py :

# Enable/disable multi-processing
OPENMP = False
OPENMP_PROC = "-fopenmp"

Puis reconstruisez :

python setup.py bdist_wheel
cd dist
pip install PygameShader-xxxxxx.whl
